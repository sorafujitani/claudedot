# plan-dry-coding

claude code plan modeで実行されることが前提です。
モードで詳細な実装コードを提示し、実際のファイル編集は行わずにレビュー可能な形で出力します。

## 実行手順

1. まず実装に必要な調査を行う
   - 関連するDBスキーマの確認
   - 既存の実装パターンの確認
   - 依存関係の調査

2. 以下の形式で出力する：

### カスタムコマンド指示
```
[タスクの簡潔な説明と要件を記載]
```

### 詳細な実装コード

#### 1. [ファイルパス]
```[言語]
[完全な実装コード]
```

#### 2. 実装のポイント
- a. [実装の重要な部分の説明]
- b. [考慮事項]
- c. [使用するパターンやライブラリ]

#### 3. 考慮事項
- パフォーマンス面の考慮
- セキュリティ面の考慮
- 保守性の考慮

#### 4. テストケースの考慮
- 想定されるテストケース
- エッジケースの処理

## 注意事項
- plan modeのまま実行し、実際のファイル編集は行わない
- コードは完全で実行可能な形で提示する
- 既存のコードパターンに従う
- 必要なインポート文をすべて含める
